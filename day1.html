Q1.Difference between “ == “ and “ === “ operators.

== and === operators in JavaScript is used for comparing two variables

== compares the two variables,but it ignores the datatype of the variable

it compares slower

            var x="5";
            var y=5;
            if(x==y){
                return "hi";
            }

it prints hi

=== compares the two variables and also compares the datatypes of the variables

it works faster

                var x="5";
                var y=5;
                if(x===y){
                    return "hi";
                }else{
                    return "sorry";
                }

it prints sorry,because x is a string and y is number.a difference in datatype

q2.What is the spread operator?

The JavaScript spread operator ( ... ) allows us to quickly copy all or part of an existing array or object into another array or object.

                function sum(a,b,c) {
                    return a+b+c;
                  }
                  var numbers=[1,2,3];
                  console.log(sum(...numbers));

q4.What is execution context

The browser's JavaScript engine then creates a special environment to handle the transformation and execution of this JavaScript code. This environment is known as the Execution Context.

Execution context is defined as the environment in which the JavaScript code is executed.

By environment, I mean the value of this , variables, objects, and functions JavaScript code has access to at a particular time.

There are two kinds of Execution Context in JavaScript: 1. Global Execution Context (GEC) 2. Function Execution Context (FEC)

q6.What is meant by first class functions

A programming language is said to have First-class functions when functions in that language are treated like any other variable. In such a language,

1. a function can be passed as an argument to other functions,

2. can be returned by another function

3. can be assigned as a value to a variable.

            const greetings = function() {
                console.log("good morning");
